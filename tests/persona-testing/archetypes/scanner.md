# Scanner

A pure browsing archetype for visitors who form fast impressions, skim headings, and bounce quickly if the site does not orient them. Scanner is not a full persona; it is a reusable behavior layer that can be combined with personas and scenarios.

## Identity Fields

Scanner does not define a human identity by itself.

It describes a browsing style:

- low patience
- fast judgment
- heading-driven
- visual hierarchy sensitive
- high bounce risk
- strongly affected by first-screen clarity

## Behavior Fields

### Default Archetype

Scanner

### Scenario Fit

Useful for:

- first visits
- lunch-break browsing
- hiring review
- skeptical evaluation
- mobile visits
- low-attention contexts

### Context

Short on time. Not hostile, but unwilling to work hard.

### Confidence Threshold

Variable by persona, but Scanner behavior raises the evidence requirement because fewer pages are visited.

### Preferred Rooms

Scanner prefers pages with:

- clear headings
- obvious next actions
- short summaries
- strong hierarchy
- clear "start here" paths

### Ignored Rooms

Scanner tends to skip:

- long essays unless the opening is excellent
- dense archives
- pages with many competing panels
- unclear experiments
- anything requiring extensive context

### First-Visit Behavior

- scans first viewport
- reads headings before body copy
- clicks one obvious next step
- bounces if the purpose is unclear
- rarely explores more than 2-4 pages unless quickly rewarded

### Search Behavior

Medium.

Scanner may use search if it is obvious and faster than browsing.

### Complexity Tolerance

Low.

### Trust Signals

- first heading explains the page
- strong summaries
- clear navigation
- fast path to relevance
- visible dates or credibility cues

### Red Flags

- unclear homepage purpose
- too many choices
- clever labels without explanation
- important content buried below dense framing
- no obvious next step

### Return Triggers

Scanner returns only if the first visit produced a clear bookmark-worthy reason.

## Evaluation Fields

### Success

The site makes its purpose and best next step obvious within seconds.

### Failure

The site may contain good material, but Scanner never reaches it.

### Likely Reflection

"I can tell this site has depth, but I need it to help me find the right door faster."

## Notes

Scanner is one of the most important archetypes for testing homepage density and first-visit orientation.
