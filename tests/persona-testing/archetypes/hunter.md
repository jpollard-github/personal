# Hunter

A pure browsing archetype for visitors who arrive with a goal and want the shortest path to useful information. Hunter is not a full persona; it is a reusable behavior layer that can be combined with clients, hiring managers, builders, subscribers, skeptics, or anyone with a specific task.

## Identity Fields

Hunter does not define a human identity by itself.

It describes a browsing style:

- task-oriented
- impatient with detours
- likely to use search
- values clear labels
- quickly ignores irrelevant rooms
- measures success by whether the goal was completed

## Behavior Fields

### Default Archetype

Hunter

### Scenario Fit

Useful for:

- potential client evaluation
- hiring manager review
- RSS subscription lookup
- project discovery
- pricing/contact lookup
- returning visitor looking for one item

### Context

Goal-driven. Often short on time. Not opposed to personality, but only after the task is satisfied.

### Confidence Threshold

Variable by persona, but Hunter behavior tends to reward fast evidence.

### Preferred Rooms

Hunter prefers:

- Search
- Work With Me
- Projects
- Build Log
- Updates
- clear contact/intake pages
- pages with direct answers

### Ignored Rooms

Hunter skips:

- long emotional context
- unrelated hobbies
- atmospheric detours
- anything not connected to the immediate task
- pages with unclear labels

### First-Visit Behavior

- scans homepage for the relevant door
- uses search early if available
- follows the most practical route
- visits fewer pages than Wanderer
- leaves when the goal is met or blocked

### Search Behavior

High.

Likely searches depend on scenario.

Examples:

- pricing
- contact
- AI
- automation
- RSS
- projects
- resume
- consulting

### Complexity Tolerance

Medium.

They tolerate detail after relevance is established.

### Trust Signals

- fast path to answer
- clear CTA
- strong headings
- concise summaries
- useful search results
- no buried essentials

### Red Flags

- unclear navigation
- hidden contact path
- too much personality before utility
- ambiguous labels
- search that does not help

### Return Triggers

Hunter returns if the site solved the task efficiently and appears reliable for future tasks.

## Evaluation Fields

### Success

Hunter finds the intended information quickly and with confidence.

### Failure

Hunter believes the site may contain the answer but is not worth digging through.

### Likely Reflection

"The site has personality, but I need it to respect my task first."

## Notes

Hunter is the primary archetype for testing Search, Work With Me, pricing/contact paths, hiring flows, and practical site utility.
