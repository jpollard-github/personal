# Builder

A reusable browsing archetype for visitors who evaluate what has actually been made. Builder is not a full persona. It is a behavior layer that can be combined with personas such as Fellow Programmer, Creative Technologist, Hiring Manager, Potential Client, or indie-maker visitors.

## Identity Fields

Builder does not define a human identity by itself.

It describes a browsing style:

- proof-oriented
- implementation-curious
- project-first
- allergic to vague claims
- interested in systems, process, and outputs
- willing to read technical detail if it demonstrates real work

## Behavior Fields

### Default Archetype

Builder

### Scenario Fit

Useful for:

- technical credibility review
- project discovery
- hiring/collaboration evaluation
- "what has this person actually built?" visits
- AI/tooling evaluation
- build-log review
- side-project or product exploration

### Context

Focused curiosity. Moderate-to-high technical patience. Low patience for personality before evidence unless the persona itself says otherwise.

### Confidence Threshold

Variable by persona.

Builder behavior usually requires visible evidence before trust rises.

High-confidence personas using Builder should be especially hard to satisfy without concrete artifacts.

### Preferred Rooms

1. Projects
2. Build Log
3. Work With Me
4. Search
5. Updates
6. About
7. technical project detail pages
8. generated systems such as Brand Kit, Persona Testing, or Dev Assistant if visible

Exact room order should be modified by persona and scenario.

### Ignored Rooms

Builder may initially skip:

- Cats
- Music, unless framed as generated/reporting system
- Twin Peaks, unless framed as an interactive or design system
- Arcade nostalgia, unless there is a game/tool artifact
- purely emotional writing
- admin pages unless the scenario is internal-tool review

### First-Visit Behavior

Typical behavior:

1. Starts on homepage or a project deep link.
2. Looks for Projects, Build Log, or Work With Me quickly.
3. Uses Search if the technical path is not obvious.
4. Checks whether projects have status, purpose, and evidence.
5. Looks for generated outputs, screenshots, code, or build notes.
6. Samples About only after technical credibility begins to form.
7. Bounces if the site feels like ideas without artifacts.

### Search Behavior

High.

Likely searches:

- AI
- dev assistant
- generator
- persona
- brand
- Vercel
- Next.js
- project
- build log
- automation
- MCP

### Complexity Tolerance

High.

Builder can handle complexity if it is structured around systems, decisions, and outputs.

Low tolerance for vague or decorative complexity.

### Trust Signals

- concrete projects
- active build log
- generated artifacts
- source links or implementation notes
- honest status labels
- realistic constraints
- technical tradeoffs
- visible iteration
- clear problem statements

### Red Flags

- many ideas with no shipped evidence
- vague project cards
- no current status
- no implementation notes
- buzzwords without decisions
- AI claims without concrete workflow
- difficult path from claim to proof

### Return Triggers

Builder returns when:

- projects ship updates
- build log shows progress
- a tool becomes reusable
- implementation notes deepen
- generated outputs improve
- new systems appear

## Evaluation Fields

### Success

The visitor thinks:

> "This person actually builds systems, not just concepts."

### Failure

The visitor thinks:

> "There are interesting ideas here, but I cannot tell what is real."

### Likely Reflection

"The site works for Builder when claims quickly connect to working artifacts, build history, and implementation reasoning."

## Notes

Builder is one of the most important archetypes for professional credibility and future product portability.

In Journey Mode, Builder should usually:

- prioritize Projects and Build Log
- use Search more often than Wanderer or Reader
- skip purely personal rooms unless the persona requires them
- require evidence before trust increases
- penalize stale or unclear project status
- reward generated outputs and explicit tradeoffs
