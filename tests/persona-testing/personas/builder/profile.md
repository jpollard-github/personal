# Builder

A maker, indie hacker, developer, or technically curious creator who comes to ArcadeGhosts because they are interested in what Jason is building. They are less interested in personality as biography and more interested in momentum, process, tooling, and whether the projects solve real problems.

## Identity Fields

### Who this person is

- indie developer
- maker
- founder
- technical hobbyist
- AI workflow experimenter
- person who learns by building

### Rough life stage

Any adult builder, likely familiar with side projects, prototypes, and the tension between ideas and shipping.

### Relationship to Jason or the site

They may find the site through an AI meetup, GitHub, build log, dev-assistant discussion, or a shared interest in tooling.

### What they value

- momentum
- proof of work
- reusable systems
- clear problem framing
- implementation detail
- iteration
- practical experiments

### Why they showed up

They want to answer:

> "What is this person building, and is any of it useful, reusable, or inspiring?"

## Behavior Fields

### Default Archetype

Builder

They head directly toward projects, build logs, and implementation notes.

### Scenario

Evaluating ArcadeGhosts as a builder's public workbench.

### Context

Curious, practical, low patience for unrelated biography until they see useful work.

### Confidence Threshold

Medium

Working artifacts build confidence fast.

### Preferred Rooms

1. Projects
2. Build Log
3. Work With Me
4. Search
5. Updates
6. About
7. Brand Generator / persona testing if visible

### Ignored Rooms

Initially:

- cats
- dating/personality content
- music unless generated from data
- Movies & TV unless tied to an app or experiment
- static nostalgia

### First-Visit Behavior

- looks for Projects immediately
- opens Build Log
- searches for AI/tooling terms
- checks whether projects have clear purpose
- follows generated systems like merch/brand/persona testing
- may ignore emotional rooms unless they frame the work

### Search Behavior

High.

Likely searches:

- AI
- generator
- persona
- brand
- dev assistant
- Vercel
- projects
- MCP

### Complexity Tolerance

High.

They like complexity if it supports a system.

### Trust Signals

- working prototypes
- clear READMEs
- generated outputs
- build logs
- automation
- thoughtful constraints
- repeatable workflows
- honest abandoned/paused projects

### Red Flags

- lots of ideas, few artifacts
- unclear status
- no recent activity
- weak project descriptions
- over-polished claims
- no explanation of what problem a tool solves

### Return Triggers

They return for project updates, code releases, generator improvements, persona testing reports, or build-log progress.

## Evaluation Fields

### What success looks like

They think:

> "This person ships systems, not just ideas."

### What failure looks like

They think:

> "There are many concepts here, but I cannot tell which ones are real."

### What they would likely say after visiting

"ArcadeGhosts is most compelling when the site shows the machinery behind the ideas."

## Notes

Builder can be used as either a full persona or an archetype. As a persona, this version represents the human who values shipped systems above all else.
