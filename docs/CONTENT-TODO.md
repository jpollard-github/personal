## How These Work Together

- `CONTENT-TODO.md` is the backlog and publishing pipeline.
- `EDITORIAL-GUIDE.md` explains how the content should feel.
- `low-friction-content-flow.md` explains how captured fragments move through the admin workflow.

## How To Use This File

1. Capture raw ideas in Content Inbox when possible.
2. Move promising items into the right `CONTENT-TODO` section.
3. Keep only a few items in Drafting at once.
4. Publish small things often.
5. Use the Editorial Guide before polishing.
6. Use persona tests only after meaningful content or navigation changes.

Reminder:

- Prefer finishing one piece of content over starting five new drafts.

## Current Focus

1. Publish new Build Log entries.
2. Publish one or two essays.
3. Grow Tiny Thoughts.
4. Improve existing rooms before creating many new ones.
5. Review near-publish content against `docs/EDITORIAL-GUIDE.md` before shipping.

Use this as the day-to-day publishing guide.

Keep the active drafting load intentionally small:

- no more than 1-2 active essay drafts
- no more than 1 active Build Log post near publishable
- a small handful of Tiny Thoughts is enough

Manual review step:

- Before publishing, review draft content against `docs/EDITORIAL-GUIDE.md` for voice, specificity, warmth, and whether it feels alive rather than generic.
